When Round 1 opens, you'll choose 1 of 4–5 problem statements and build an OpenEnv environment around it.

"Build a mini-game RL environment with clearly defined tasks, automated graders, and reward logic using the OpenEnv framework."

→ Write graders that verify task completion

→ Define reward logic for scoring

→ Package using OpenEnv for automated evaluation

 This video is a comprehensive bootcamp for the **Meta PyTorch OpenEnv Hackathon**, hosted by the **Scaler School of Technology**. It features **Ben Burtenshaw** from **Hugging Face** and **Pulkit Aneja**, focusing on how to build and deploy Reinforcement Learning (RL) environments using the **OpenEnv** library.

-----

### \#\# Key Topics and Timestamps

  * **Introduction to RL & OpenEnv [06:41](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=401):** Ben explains the core concepts of Reinforcement Learning, emphasizing that a "good" environment represents a real-world task that could be used in post-training runs for LLMs.
  * **The RL Workflow [10:02](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=602):** A walkthrough of how an LLM generates candidates (e.g., code), which are then tested in an environment to receive a **reward signal** (score).
  * **OpenEnv (OpenM) Library [30:18](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=1818):** Ben introduces **OpenEnv** (short for environment), a collaboration between Meta, PyTorch, Hugging Face, and Unsloth. It provides a single API for creating interoperable RL environments.
  * **Building Your First Environment [41:39](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=2499):** A live demo using the CLI command `openm init` to generate a project skeleton, including the action, observation, and state models.
  * **Using AI Agents for Coding [42:50](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=2570):** demonstrating how to use agents like **Codeex** with "OpenM skills" to automatically generate environment code from natural language descriptions.
  * **Step-by-Step Submission Walkthrough [59:54](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=3594):** Pulkit demonstrates the practical steps for the hackathon:
      * Setting up a virtual environment.
      * Using **Docker** to build and run the environment locally.
      * Configuring the **Inference Script** (mandatory for evaluation).
      * Deploying to **Hugging Face Spaces** using `openm push`.

-----

### \#\# Important Hackathon Guidelines

>   * **No Toy Environments [01:05:40](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=3940):** Submissions must simulate **real-world tasks** (e.g., healthcare triage, Git operations, kernel optimization) rather than simple games like Wordle or Echo.
>   * **Hugging Face Tokens [01:30:20](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=5420):** Participants do **not** need a paid OpenAI API key. Use a Hugging Face token with the provided "router" in the inference script.
>   * **Mandatory Inference Script [01:13:05](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=4385):** The script is essential for the judges to evaluate the quality and utility of your environment.
>   * **Deadline [01:44:34](http://www.youtube.com/watch?v=kkCNMz0Ptd8&t=6274):** Round 1 submissions close on **April 8th**.

Functional Requirements
1. Real-World Task Simulation
The environment must represent tasks that humans actually perform in real settings—no games or toy problems.
Examples include email triage, code review, data cleaning, scheduling, customer support, and content moderation.

2. OpenEnv Specification Compliance
The environment must fully implement the OpenEnv interface, including:
Typed Observation, Action, and Reward models using Pydantic
step(action) → returns (observation, reward, done, info)
reset() → returns the initial observation
state() → returns the current state
An openenv.yaml file containing metadata
The implementation must successfully pass validation via openenv validate.

3. Minimum of Three Tasks with Agent Graders
Provide at least three tasks, each with a clearly defined objective
Tasks should span increasing difficulty: easy → medium → hard
Each task must include a programmatic grader that assigns a score between 0.0 and 1.0
Grading criteria must be clear, deterministic, and reproducible

4. Meaningful Reward Function
The reward function must provide feedback throughout the task trajectory, not just at completion
It should reward incremental progress toward the objective
It must penalize undesirable behaviors such as infinite loops or destructive actions

5. Baseline Inference Script
Include an inference script that uses the OpenAI API client to evaluate a model within the environment
API credentials must be read from environment variables (HF_TOKEN)
The script should produce a reproducible baseline score across all tasks

Non-Functional Requirements
1. Deployment on Hugging Face Spaces
The environment must be deployable as a containerized Hugging Face Space
It should be tagged with openenv

2. Containerized Execution
Provide a working Dockerfile
The environment must build and run successfully using:
docker build
docker run

3. Documentation
The README must include:
Environment overview and motivation
Definitions of action and observation spaces
Task descriptions with expected difficulty levels
Setup and usage instructions
Baseline performance scores
